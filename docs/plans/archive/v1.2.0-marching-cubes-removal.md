# v1.2.0 - marching-cubes removal

## Mission

Remove marching cubes from the app completely. The product direction is no longer
"default-off surface extraction"; it is particle/liquid-cell scale. Done means there
is no Mesh button, no mesh URL toggle, no MC settings, no WASM mesh bridge, no lazy MC
GPU allocation, no MC shaders/runtime module/tests, and no architecture or decision
doc claiming MC is a supported path.

This phase comes first because MC pollutes performance measurements, settings
organization, and visual direction.

## Scope

In scope:

- Delete the web Mesh button and `?mesh=1` behavior.
- Remove `set_mesh_enabled`, `set_mesh_iso`, and every MC-only WASM export.
- Remove settings whose only effect is MC: `dev.mesh_enabled`, `render.mesh_*`, and
  `render.sun_*` unless an implementer deliberately repurposes sun direction for
  particle lighting in a separate plan.
- Remove MC runtime ownership from `gpu/mod.rs`: `mesh::MeshExtractor`,
  `MeshLook`, lazy allocation, offscreen scene-color/refraction/back-depth/blit path
  if it is MC-only, and MC memory accounting.
- Delete MC-only modules and shaders after confirming no non-MC path uses them:
  `gpu/mesh.rs`, `gpu/blit.rs`, `gpu/shaders/{mc,mc_args,mesh,density,blur,blit}.wgsl`.
- Remove host marching-cubes tables/tests (`sim/marching_cubes.rs` and its module
  export) unless a test is still needed for a non-MC path. The default is deletion.
- Rewrite docs to state that render modes are particles, tank, and grid/liquid-cell
  inspection; MC is removed, not merely hidden.

Out of scope:

- Designing a replacement surface renderer.
- Adding new particle shading, density splats, screen-space metaballs, or other visual
  substitutes. Those belong to v1.3.0 if profiling says they help scale perception.
- Keeping MC as archived live code. The user explicitly chose gutting it from
  everything.

## Approach

Work sequentially; this touches central Rust, shaders, web, and docs.

1. **Surface inventory.** Run targeted search for `mesh`, `marching`, `mc_`,
   `density`, `blur`, `blit`, `render.sun`, and `dev.mesh_enabled`. Record every anchor
   in the agent report so the orchestrator can verify no product path remains.
2. **Web and bridge removal.** Remove `#btn-mesh`, Mesh button styling/state, URL
   `?mesh=1`, `app.set_mesh_enabled`, and public mesh setters from the WASM class.
3. **Settings removal.** Delete MC-only settings and accessors. Verify `config_json`
   no longer emits MC controls. If sun direction only drives `mesh.wgsl`, remove those
   settings too; do not leave dead sliders.
4. **GPU runtime removal.** Remove `mod mesh`, `mod blit`, MC fields, offscreen
   water targets, lazy allocator, extraction pass after substeps, MC render branch, MC
   setters, and MC buffer memory contributions. Keep the ordinary tank/particle/slice
   render path intact.
5. **Source/test deletion.** Delete MC-only modules/shaders and update `sim/mod.rs` and
   host tests. Host tests should still cover grid indexing, pressure, and simulation
   math.
6. **Docs migration.** Rewrite current-state docs in place and remove MC policy entries
   that no longer apply. Add a durable rendering decision: particle/voxel views are the
   product surface; old MC was retired because it distracted from scale and carried
   heavy memory/rendering cost.

## Discipline rules

- Do not keep a hidden dev MC path.
- Do not replace MC with a new surface technique in this phase.
- Do not remove particle rendering, tank wireframe, grid slice, profiler, or capture
  harness behavior.
- When deleting code, update architecture/decisions in the same implementation wave so
  a fresh chat does not rediscover dead MC docs.

## Migration notes

- `architecture/rendering.md` - now documents only the single-pass particle,
  tank-wireframe, and grid-slice renderer.
- `architecture/gpu-resources.md` - removed lazy extraction resources, offscreen water
  targets, and extracted-surface memory accounting.
- `architecture/settings.md` - removed all surface-only, water-look, and sun controls;
  retained and documented particle-look controls.
- `architecture/web-shell.md` and `architecture/app-shell.md` - removed the Mesh
  toolbar/URL/bridge and tank-local surface-camera contract.
- `decisions/rendering.md` - records particle/liquid-cell views as the product surface
  and the single-pass rendering policy.
- `decisions/performance.md` - records that dormant extracted-surface cost is not
  carried in scale measurements.
- `decisions/scope.md`, `decisions/platform.md`, and both routing indexes - removed
  current-state claims that the retired path exists.
