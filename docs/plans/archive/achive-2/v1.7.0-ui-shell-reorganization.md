# v1.7.0 - UI shell reorganization

## Mission

Make the browser shell feel like one coherent product surface: quiet transparent
header chrome, a right-side tabbed config/profiler workspace, and a bottom product-mode
launcher that starts in Auto Rotate. Done means the default page loads with no panels
open, Auto Rotate active, the profiler reachable through the workspace tab, and no raw
interaction enable toggles or config-copy affordance exposed to users.

## Scope

In scope:

- Combine the header cleanup, config workspace tabs, and bottom launcher work into one
  UI stream.
- Move the config/profiler workspace to the **right** side. It should not have a left
  placement mode.
- Replace separate Config and Profiler panels with one workspace containing tabs:
  Render, General, Physics, Modes, and Profiler.
- The cog opens the workspace on **General** every time. Do not restore the last active
  tab, and do not add tab-routing metadata.
- Panels/tabs start closed on initial load. Fix the startup bug where panels are open
  or flash open during initialization.
- Remove or neutralize existing `?panels=on` auto-open behavior. Do not add a
  query-param path for opening a specific tab. Existing capture defaults should show
  Auto Rotate mode with tabs closed.
- Keep Advanced and Dev controls as per-tab collapsed groups where they apply.
- Make the header transparent/see-through over the stage.
- Show the Fluid Lab title with the version sourced from `web/package.json` or the
  package metadata that writes it. Do not hardcode a second version label.
- Replace text toolbar commands with accessible icon buttons for pause/play, reset,
  and config.
- Remove the standalone Profiler toolbar button after Profiler is a workspace tab.
- Remove the Copy Config JSON product affordance and any copy-specific helper path.
  Keep the `config_json` bridge if the rendered config panel still needs it.
- Add the bottom product-mode launcher:
  - Auto Rotate
  - Waves
  - Manual
- Default first load to Auto Rotate.
- Reset preserves the selected bottom mode. Page reload returns to Auto Rotate because
  bottom mode is not saved to localStorage.
- Auto Rotate and Waves are mutually exclusive. Manual disables scheduled Auto Rotate
  and Waves behavior.
- Manual reveals the existing pointer modes: camera, rotate, rotate/roll, and slosh.
- Number keys target manual pointer modes only; they do not become product-mode
  shortcuts.
- Remove raw user-facing enable toggles for auto-roll and waves. Keep mode-specific
  values visible, clearly grouped under the mode they affect.

Out of scope:

- New wave patterns, auto-roll presets, randomness changes, source/drain, or new
  physics.
- New high-count presets.
- Replacing the settings registry as the config source of truth.
- Rewriting tooltip/technical help copy except where labels must match the new layout.
- Architecture or decisions doc migration before implementation work is underway.

## Approach

Treat the UI as one state machine:

- Header owns global commands: pause/play, reset, config workspace.
- Workspace owns config/profiler visibility and tabs. It always opens on General.
- Bottom launcher owns product interaction mode: Auto Rotate, Waves, or Manual.
- Rust remains the owner of scheduled interaction behavior. JavaScript may select the
  mode and set existing scheduler values, but it must not create a parallel scheduler.

Implementation should start by defining the JS state transitions for workspace
visibility and bottom mode. After that, re-home panel rendering into right-side tabs,
then simplify the header and bottom controls against that state model.

If raw enable booleans still need to exist internally to drive Rust scheduling, they
should not render as user config rows and should not persist as user choices. The
product mode is the user-facing enable state.

### State machine detail

| State/input | Source of truth | Required transition | Persistence / edge case |
|---|---|---|---|
| Workspace open/closed | JS shell state for the single right-side workspace. | Initial load is closed before first paint. The config cog toggles the workspace; closed -> open always opens on General. Remove/neutralize `?panels=on` auto-open. | Reset does not open or close it. Reload returns closed. Do not persist workspace visibility in localStorage or URL state. |
| Active workspace tab | JS shell state scoped to the open workspace. | No visible tab while closed. User tab clicks select Render, General, Physics, Modes, or Profiler while open. Every cog open/reopen lands on General rather than the previous tab. | Reset does not retarget an already open tab. Reload has no saved tab. Do not add tab URL metadata or localStorage. General is always present. |
| Bottom product mode | JS product-mode state that writes existing Rust scheduler settings as needed. | Initial load selects Auto Rotate. Launcher selects exactly one of Auto Rotate, Waves, or Manual. Auto Rotate enables auto-roll behavior and disables waves; Waves disables auto-roll and enables waves; Manual disables both scheduled behaviors. | Reset preserves the selected product mode and reasserts scheduler values if a reset-class rebuild makes that necessary. Reload returns to Auto Rotate because no product-mode state is stored. |
| Manual pointer mode | Existing manual pointer sub-state under Manual. | Default is camera. Manual reveals camera, rotate, rotate/roll, and slosh. Buttons and number keys 1-4 target only these manual pointer modes; number keys are not product-mode shortcuts. | Reset preserves the current manual pointer sub-mode. Reload returns it to camera. Do not store it in localStorage. |
| Reset command | Header reset button and `r` keyboard path call the existing Rust reset. | Rebuild/reset the sim through `app.reset()` and keep current workspace visibility, active tab, bottom product mode, and manual pointer sub-mode. Re-render setting badges as the current panel does. | Reset applies pending Reset-class settings but does not choose a product mode for the user. |
| Page reload | Fresh JS state plus restored config settings. | Start workspace closed, no visible active tab, bottom product mode Auto Rotate, and manual pointer mode camera. | Settings may restore from localStorage; workspace/tab/product/pointer shell state may not. |
| localStorage | Existing settings persistence under `fluidlab.config.v1`. | Config row edits continue to persist actual settings and preserve reset/reload badge behavior. Bottom-mode clicks must not write a new product-mode key or make hidden auto-roll/wave enable booleans durable user choices. | On startup, restore settings before applying the Auto Rotate product-mode transition so stale stored scheduler values cannot make Waves or Manual appear selected. |

## Exit gate

- Default browser load shows Auto Rotate mode and no open config/profiler workspace.
- There is no startup frame where panels visibly open during initialization.
- Cog opens the right-side workspace on General; reopening also lands on General.
- Render, General, Physics, Modes, and Profiler tabs are present.
- Profiler data still renders from `stats_json` on the existing cadence.
- Settings still render from `config_json`, apply through `set_setting`, preserve
  reset/reload badges, per-setting defaults, and localStorage for actual settings.
- Raw auto-roll/wave enable toggles are not visible as user config controls.
- Reset preserves the selected bottom mode; reload returns to Auto Rotate.
- Auto Rotate and Waves cannot both be active.
- Manual shows the four pointer modes, and number keys affect only manual pointer
  modes.
- Header shows version from package metadata and uses accessible icon buttons.
- Standalone Profiler toolbar button and Copy Config JSON affordance are gone.
- Real browser captures verify:
  - default desktop load: tabs closed, Auto Rotate active;
  - scripted workspace open: right-side General tab, no overlap;
  - scripted Manual mode: manual pointer controls visible, number keys change only the
    manual pointer sub-mode, not the product mode;
  - scripted reset from Waves or Manual: selected bottom mode and workspace state
    survive `app.reset()`;
  - scripted reload/localStorage: after a normal persisted setting change and a
    bottom-mode change, hard reload returns to Auto Rotate with tabs closed, restores
    the normal setting, and has no bottom-mode storage key;
  - narrow width: header, workspace, and bottom launcher do not overlap.

## Discipline rules

- Do not hide manual camera orbit. It remains the safe inspection mode.
- Do not add a JS interaction scheduler.
- Do not use localStorage for bottom product mode.
- Do not remove `config_json` unless the config panel has a replacement bridge.
- Do not add URL state for tabs.
