# v1.6.0 - interactive forces

## Mission

Add low-risk motion tools that make the particle/liquid-cell app feel alive after the
scale and wall-quality work: random tank auto-rotation and impulse-based wave making.
Done means the app can roll/slosh itself in varied ways without reintroducing wall
stickiness, and source/drain has been moved to the future roadmap rather than mixed
into this phase.

## Scope

In scope:

- Random tank auto-rotater that rotates the tank in varied ways. The first version can
  be simple/random; a later pass can encode the user's preferred "best water physics"
  rolling patterns.
- Impulse wave maker based on existing impulse/slosh plumbing.
- Settings or simple controls for enabling/disabling, intensity, cadence, and maybe
  seed/randomness if useful.
- Capture showing the tools running on the post-v1.4.0 wall model.

Out of scope:

- Source/drain implementation. Future source/drain should create and destroy
  water/particles, but it needs its own mass-mutation plan.
- Dynamic solid objects.
- Open boundaries or ocean/heightfield waves.
- MC or surface extraction work.

## Approach

### Wave A - auto-rotation product shape

- Decide whether the control lives in settings, the toolbar, or a compact presentation
  control. Prefer settings if it needs parameters; prefer toolbar only for a clear
  on/off command.
- Define a random tank-roll schedule: axis selection, angular speed/range, dwell time,
  easing/noise, and reset behavior.
- Make camera auto-rotation separate or absent. The user asked for tank rotation, not
  camera spin as the main feature.

### Wave B - impulse wave maker

- Reuse the existing impulse pass where possible.
- Add a periodic or triggerable impulse pattern with strength/frequency/direction
  controls if those are product-useful.
- Keep it a force/impulse tool, not a physical wall paddle.
- Confirm it composes with random tank roll without destabilizing the sim.

### Wave C - source/drain future note

- Update `docs/plans/roadmap.md` to keep source/drain as future work.
- Future source/drain direction: create/destroy particles or water volume, not merely a
  visual trick. It should get a separate plan because it touches particle allocation,
  mass accounting, classification, and possibly boundary semantics.

## Exit gate

- Random tank auto-rotation works and can be stopped/reset.
- Impulse wave maker works and has clear settings/controls.
- Dam-break/wall-contact behavior from v1.4.0 does not visibly regress during a
  real-GPU capture.
- Source/drain is absent from the shipped UI and present in `roadmap.md` as future
  work.
- `cd /home/adamg/fluid-simulation/app && cargo build --target wasm32-unknown-unknown`
  passes.
- `cd /home/adamg/fluid-simulation/app && cargo test --lib` passes.

## Discipline rules

- Do not implement source/drain in this phase.
- Do not use random roll to hide wall cling; wall behavior must already be fixed.
- Do not add a physical wall paddle unless the plan is rewritten.
- Keep controls simple enough that v1.5.0 settings organization still makes sense.

## Ship record

Implemented controls:

- `interaction.auto_roll_enabled`
- `interaction.auto_roll_strength`
- `interaction.auto_roll_cadence`
- `interaction.wave_enabled`
- `interaction.wave_strength`
- `interaction.wave_frequency`

Runtime behavior:

- Auto-roll advances only while Running, rotates the tank (not the camera), pushes
  gravity after pose changes, and resets schedules on Reset without disabling settings.
- Wave maker advances only while Running and emits alternating local horizontal
  velocity impulses via the existing one-shot GPU impulse pass.
- Pause freezes both schedules; source/drain remains absent.
