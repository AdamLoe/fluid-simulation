# v1.12.0 — Hero water: scene-color refraction (+ shared foundation)

> First plan in the **hero-water series** (v1.12→v1.18). This one is special: besides
> refraction it lays the foundation every later plan reuses. Read the *Cross-cutting
> decisions* block below — later plans reference it instead of restating it.

## Mission

Add **scene-color refraction** to the existing screen-space water composite so the tank
floor and backdrop visibly bend through the liquid, and stand up the shared hero-water
foundation: a typed render-mode enum, a **Water** settings tab of Live controls, a
scene-color/scene-depth offscreen prepass, a minimal **refractable environment**
(textured floor + backdrop + matte walls), and a hero-water debug-view selector. Done
when the calm-fill capture shows the floor/backdrop refracting through the water, the
result beats both the simple-particle baseline and the current non-refractive composite,
and `refraction_strength` visibly changes the bend live via the slider.

This is the highest-value / lowest-risk increment: the composite in
[`composite.rs`](../../app/crates/fluid-lab/src/gpu/composite.rs) already has thickness,
smoothed front-depth, a reconstructed normal, Fresnel, and Beer-Lambert absorption.
Refraction is the missing piece [`decisions/rendering.md`](../decisions/rendering.md)
explicitly deferred ("Refraction requires a separate scene-color target and visible
scene detail to justify that cost"). We pay that cost here.

## Cross-cutting decisions (the whole series inherits these)

1. **Evolve the existing screen-space composite into the hero path — do NOT build a
   parallel `HeroWater` pipeline.** The ChatGPT source plan invented a 5th `RenderMode`;
   that duplicates a renderer that already does most of the material. Hero features
   (refraction, foam, marching-cubes surface, env reflection, caustics, wet walls,
   temporal) are **Live-toggleable sub-features of the water view**, each with its own
   `enabled` flag — not new top-level modes.
2. **Replace the bare `u32 particle_view` dispatch with a typed `RenderMode` enum.**
   Today [`gpu/mod.rs`](../../app/crates/fluid-lab/src/gpu/mod.rs) matches on
   `self.particle_view` with `PARTICLE_VIEW_*` consts. Replace with
   `enum RenderMode { Water, OpticalParticles, SimpleParticles }` (slice overlay stays
   orthogonal). `render.particle_view` keeps mapping 0/1/2 to these for compatibility;
   the optical/simple views remain the explicit fallbacks the series must never break.
3. **New `Category::Water` + a "Water" tab.** Add the variant in
   [`settings/mod.rs`](../../app/crates/fluid-lab/src/settings/mod.rs) and one
   `TAB_CATEGORY_ALLOWLIST` entry (`water: new Set(["Water"])`) in
   [`web/panels.js`](../../app/web/panels.js). Every hero setting is
   **`ApplyClass::Live`** so sliders auto-apply with no reset. Use `PanelGroup` /
   per-feature sub-headers to keep ~150 controls navigable. The volume is acceptable
   (user confirmed) as long as it stays Live.
4. **One `HeroWaterParams` uniform, rebuilt from the registry each frame.** Mirror the
   Water settings into a single std140-aligned uniform every frame (cheap). Sliders are
   live without pipeline rebuilds. No per-setting GPU plumbing.
5. **A `render.hero.debug_view` u32 enum** routes a debug blit to the swapchain. Each
   plan appends its own views.
6. **No normal-frame readback** (unchanged invariant). Every hero pass is GPU-side;
   any stat counts go through the throttled
   [`timing.rs`](../../app/crates/fluid-lab/src/gpu/timing.rs) path only.
7. **Acceptance is the capture harness.** Every visible/GPU change needs a
   `tools/capture.mjs` run against a fixed scene + a `gpu.render_ms` profiler number,
   built/tested **inside WSL** (see [`agent-context/build-run.md`](../agent-context/build-run.md)).
   Never claim a perf result without profiler output.

## Scope

**In:**
- `RenderMode` enum refactor (behavior-preserving; capture parity vs current main).
- `Category::Water` + Water tab + `HeroWaterParams` uniform plumbing.
- `scene_color` (`Rgba16Float`) + `scene_depth` offscreen targets. Render wireframe +
  environment into them *before* the water passes, then blit `scene_color`→swapchain so
  non-refracted background still shows where there's no water.
- **Refractable environment** (minimal but real): textured tank floor, gradient/grid
  backdrop, matte walls — enough scene detail for refraction to be visible.
- Refraction in `composite.wgsl`: `refract_uv = uv + N.xy * strength * thickness_factor`,
  clamp to `refraction_max_offset_px`, invalid-sample fallback to unrefracted
  `scene_color`, and a depth guard so geometry in *front* of the water isn't smeared.
- Re-tune Beer-Lambert absorption now that there's a real background behind the water.
- Debug views: `SceneColor`, `SceneDepth`, `Thickness`, `RefractionUvOffset`, `Fresnel`,
  `Absorption`, `FinalWaterOnly`.

**Out (later plans):** foam/diffuse particles (v1.13), marching-cubes surface (v1.14),
**reflected** procedural sky/room (v1.15 — this plan only builds the *refracted
background*, not the reflected environment map), caustics (v1.16), wet walls/meniscus
(v1.17), temporal stabilization (v1.18). Full PBR walls are out; the environment here is
minimal-but-textured.

## Approach

Single serial stream — it touches `gpu/mod.rs::render`, `composite.rs`/`.wgsl`, the
settings registry, and the web shell, which don't parallelize cleanly.

1. **RenderMode refactor.** Swap the `u32`/consts for the enum. Capture before/after on
   all three views to prove zero visual change.
2. **Water tab + empty params.** `Category::Water`, panels.js tab, `HeroWaterParams`
   uniform wired but unused. Confirm a placeholder Live slider round-trips.
3. **Scene prepass.** Create `scene_color`/`scene_depth`; draw wireframe + environment
   into them; blit to swapchain. Reuse the existing shared depth texture for
   `scene_depth` rather than adding a second depth attachment if formats allow.
4. **Environment shaders.** Floor texture (procedural grid/checker), backdrop gradient,
   matte wall tint — all into `scene_color`.
5. **Refraction.** Sample `scene_color` at `refract_uv` in `composite.wgsl`; add the
   settings + debug views.
6. **Capture + tune + profiler.** Scene 1 (calm fill); record `gpu.render_ms` delta.

## Watch-outs

- **The composite currently blends directly over the swapchain** (no scene-color target;
  [`rendering.md`](../architecture/rendering.md) §Screen-space water). Introducing
  `scene_color` changes the pass order: environment→`scene_color`→blit to swapchain→water
  composites over swapchain while *sampling* `scene_color` for the refracted tap. Get
  this order right or the wireframe/background disappears where there's no water.
- **Refraction is invisible without scene detail** — that's exactly why the textured
  environment is in *this* plan and not deferred to v1.15.
- **Bounded tank → small offsets.** Clamp in pixels; an unclamped `N.xy` offset reads as
  a warped mess at grazing angles.
- `f0` is derivable from `ior` (Schlick); expose `ior` and compute `f0`, don't store both
  as independent free knobs that can disagree.

## Settings (Water tab → Refraction / Material group, all Live)

`render.hero.mode_enabled`, `render.hero.debug_view` (enum), then:
`ior`, `refraction_strength`, `refraction_thickness_scale`, `refraction_max_offset_px`,
`invalid_refraction_fallback` (enum), `absorption_color.{r,g,b}`, `absorption_strength`,
`base_tint.{r,g,b}`, `transparency`, `deep_water_darkening`, plus environment knobs
`floor_pattern_scale`, `floor_pattern_strength`, `backdrop_strength`, `wall_visibility`.
(Names map to the ChatGPT `[render.hero_water.material]`/`.environment` blocks; flatten to
dotted ids under `Category::Water`.)

## Exit gate

- `tools/capture.mjs` on Scene 1 (calm fill) shows floor/backdrop bending through the
  water; side-by-side beats simple-particles and the pre-refraction composite.
- `render.hero.refraction_strength` slider changes the bend live (no reset).
- `gpu.render_ms` recorded before/after; delta noted in the migration section.
- All three render modes still render correctly (no regression to optical/simple).

**Engineering deviations from the source plan (all within its granted judgment):**

- **No separate blit pass.** Instead of `environment → scene_color → blit to swapchain
  → water alpha-blends over swapchain`, the composite is **opaque** (`REPLACE`): it
  samples `scene_color` itself and writes the final pixel — the unrefracted background
  where there is no water. Same result, one fewer pass.
- **`scene_depth` is its own `R16Float` target** storing positive eye distance
  (`clip.w`, matching the thickness pass), not a re-used sample of the `Depth32Float`
  attachment (which lacks `TEXTURE_BINDING`). The depth guard compares it against the
  smoothed water front.
- **`mode_enabled`** gates refraction (forces offset → 0) rather than swapping in a
  wholly separate non-refractive composite; that is also the non-refractive comparison.

**Exit gate results:**

- WSL `cargo build --target wasm32-unknown-unknown` and `cargo test --lib` (27 tests) green.
- `tools/capture.mjs` on the default scene shows the checkered floor refracting through
  the water; the `Refraction UV offset` debug view confirms per-pixel offsets across the
  surface.
- `render.hero.refraction_strength` / `debug_view` applied live via the harness
  (`set_setting` returned `true`, no reset).
- `gpu.render_ms` ≈ 0.27 ms at 1280×800 with refraction on (negligible vs the ~1.8–2.2 ms
  pressure solve); no measurable render-cost regression.
- All three render modes still render (Water default; optical/simple unchanged).
