# v1.15.0 — Hero water: environment reflection from a procedural sky/room

> Hero-water series plan 4 of 7. Inherits the *Cross-cutting decisions* in
> [`v1.12.0-hero-water-refraction.md`](v1.12.0-hero-water-refraction.md).

## Mission

Give the water a believable **reflected environment**: a procedural sky/room gradient
sampled by the surface reflection vector, mixed into the composite via Fresnel so glancing
angles go bright and reflective. Add roughness (velocity-, normal-variance-, and
foam-driven) plus a specular sun highlight and optional micro-normals. Done when the
calm-fill capture shows bright Fresnel edges and a plausible reflected room/sky on the
surface, tunable live.

**Distinction from v1.12:** v1.12 built the *refracted background* (floor/backdrop seen
*through* the water). This plan builds the *reflected environment* (sky/room seen *on*
the water). They can share one procedural-room function but they're different taps in the
material: `mix(refracted_scene * absorption, reflected_env, fresnel)`.

## Scope

**In:**
- A procedural sky/room environment function (gradient + soft room boxes/horizon),
  sampled by `reflect(-V, N)` with roughness-based softening.
- Fresnel mix already exists in the composite — wire `reflected_env` into it.
- Roughness model: `roughness_base` + velocity scale + normal-variance scale + foam scale.
- Specular sun highlight (`sun_direction`, `sun_intensity`).
- Optional micro-normals (procedural high-freq normal perturbation, velocity-scaled) for
  surface "tooth."
- Debug view: `Reflection` / `EnvOnly`.

**Out:** image-based lighting from a real cubemap/HDRI (procedural only here); reflection
of the *actual* tank geometry/particles (screen-space reflections are a separate, heavier
project — not in this series).

## Approach

Composite-shader-local change plus a small environment module. Largely independent of
v1.13/v1.14; benefits from better normals if v1.14 mesh source is on, but works on the
screen-space normal too.

1. Procedural environment function (shareable with v1.16 caustics light direction).
2. Reflection tap + Fresnel mix in `composite.wgsl`.
3. Roughness + specular + micro-normals.
4. Capture Scene 1 (calm fill) + Scene 3 (dam break); tune; profiler.

## Watch-outs

- **Over-bright Fresnel** is the classic failure — clamp and tune `reflection_strength`
  so edges read as water, not chrome.
- Micro-normals can reintroduce the shimmer the v1.18 temporal plan is meant to fix;
  keep `micro_normal_strength` conservative until temporal lands.
- Reflected env should respond to the tank's baked rotation correctly — `V`/`N` are in
  tank-local space (model matrix is folded into `view_proj`,
  [`rendering.md`](../architecture/rendering.md)); rotate the env sample into world space
  if the reflection should stay fixed to the room rather than the tank.

## Settings (Water tab → Reflection/Material group, all Live)

Mirror `[render.hero_water.material]` reflection/roughness fields + the env block:
`reflection_strength`, `environment_strength`, `environment_mode` (enum),
`environment_rotation`, `environment_brightness`, `skybox_enabled`,
`roughness_base`, `roughness_velocity_scale`, `roughness_normal_variance_scale`,
`roughness_foam_scale`, `specular_strength`, `sun_direction.{x,y,z}`, `sun_intensity`,
`micro_normal_enabled`, `micro_normal_strength`, `micro_normal_scale`,
`micro_normal_velocity_scale`.

## Exit gate

- WSL build + `cargo test --lib` green.
- Scene 1 capture: bright Fresnel edges + plausible reflected room/sky; beats the
  refraction-only v1.12 look.
- Reflection/roughness/sun sliders change the look live.
- `gpu.render_ms` delta recorded.

**Engineering deviations from the source plan (within granted judgment):**

- **The world background is a fullscreen procedural skybox, not a backdrop quad.** This
  satisfies the added requirement that the world background not rotate with the box: the
  skybox (and the water reflection) sample `env_sample` in world space via a camera-only
  `eye_to_world` rotation, so they follow the camera but stay fixed under box rotation
  (box rotation only re-aims gravity). The old gradient backdrop quad was removed;
  `render.hero.backdrop_strength` is now an inert legacy id (superseded by
  `environment_brightness`).
- **The reflection vector is rotated to world space in the composite**, exactly as the
  watch-out anticipated — the screen-space normal is already box-independent, so only the
  camera rotation is threaded in (new `Cam` uniform, binding 8).
- **The right (+x) wall is removed** alongside the already-open front (+z), forming a clear
  viewing corner (separate decision in `decisions/rendering.md`).
- **Roughness** is driven by base + the speed-weighted whitewater fraction (velocity proxy)
  + a depth-Laplacian chop term + foam; there is no per-pixel velocity in the composite.
- **Micro-normals default off** (`micro_normal_enabled = 0`) per the shimmer watch-out.

**Exit gate results:**

- WSL `cargo build --target wasm32-unknown-unknown` + `cargo test --lib` (27 tests) green.
- `tools/capture.mjs` on the calm fill shows the reflected sky + bright Fresnel rim on the
  water, the checker floor refracting through it, and the open viewing corner; the
  `Reflection` debug view confirms the per-pixel reflected environment.
- Box-rotation capture confirms the decoupling: the tank tips ~35° (floor/walls/wireframe
  rotate, water pools toward the new down) while the sky horizon stays perfectly level.
- `render.hero.reflection_strength` / `environment_mode` / `debug_view` applied live via the
  harness (no reset).
- `gpu.render_ms` ≈ 0.50 ms at 1280×800 with reflection + skybox on (vs ~0.27 ms pre-v1.15;
  +~0.2 ms, negligible vs the ~1.8 ms pressure solve). No measurable regression.
- All three render modes still render (Water default; optical/simple unchanged).
