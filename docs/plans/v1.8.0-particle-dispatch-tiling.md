# v1.8.0 - particle dispatch tiling

## Mission

Raise the legal particle-count ceiling by replacing one-dimensional particle-linear
dispatch assumptions with a coordinated tiled dispatch model. Done means the app either
submits valid work above the current 1D ceiling and records a fresh scale matrix, or it
records the exact remaining architectural blocker without weakening preflight safety.

## Scope

In scope:

- Use [`particle-dispatch-audit.md`](particle-dispatch-audit.md) as the path inventory.
  If the audit is incomplete, complete it as this plan's first phase.
- Implement a coordinated tiled/2D dispatch indexing strategy for particle-linear
  paths.
- Update every affected shader/runtime path consistently: mark/classify inputs,
  scatter, G2P, advect/recover, impulse/wave forces, and any dispatch/preflight code.
- Preserve correct bounds checks and partial-tile behavior.
- Preserve wall-aware G2P behavior while touching G2P indexing.
- Update particle-scale preflight to reflect the new legal dispatch model and remaining
  storage/adapter limits.
- Re-run a real-GPU scale matrix after the change. Include requested 2M, 4M, and 8M if
  the adapter can attempt them safely.
- Keep all performance claims hardware-dependent and tied to raw capture/profiler
  evidence.

Out of scope:

- New particle-count presets.
- Render decimation/LOD. That remains in [`future-roadmap.md`](future-roadmap.md).
- Water-look rendering experiments.
- Source/drain or new physics features.
- Optimizing the measured bottleneck after tiled dispatch; that belongs to
  [`v1.9.0-particle-performance-followup.md`](v1.9.0-particle-performance-followup.md)
  unless the win is trivial and directly evidenced.

## Approach

Treat this as correctness first and measurement second.

Phase 1: finish the particle dispatch audit and lock the tiled indexing contract.
The contract is currently owned by
[`particle-dispatch-audit.md`](particle-dispatch-audit.md): every particle-linear
shader computes the particle index from 2D workgroup coordinates, keeps
`@workgroup_size(64, 1, 1)`, and guards `p >= params.dims.y` before touching the
particle buffer.

Phase 2: implement tiled dispatch across all particle-linear paths in one coordinated
change. A local shader fix is not sufficient because inconsistent indexing can corrupt
particle/grid transfers. Required paths are mark, scatter U/V/W, G2P, and the
standalone impulse path used by both slosh and the wave-maker. Cell/face/grid-linear
passes stay 1D unless implementation evidence shows they hit an adapter limit.

Phase 3: update preflight and stats reporting so rejected configurations remain
truthful. The old 1D workgroup ceiling should no longer be the limiting rule when
tiled dispatch is legal, but storage-binding limits, per-dimension workgroup limits,
and WGSL `u32` indexability still matter. Do this after the tiled shader/runtime paths
exist so preflight is never loosened ahead of valid submission code.

Phase 4: run the fresh scale matrix and update
[`v1.9.0-particle-performance-followup.md`](v1.9.0-particle-performance-followup.md)
with the measured top bottleneck.

## Implementation surfaces

Use the audit's "Required code surfaces for v1.8.0" table as the implementation
checklist. In short:

- `crates/fluid-lab/src/gpu/fluid.rs` owns the shared particle dispatch shape and
  every dispatch call that must become tiled.
- `mark.wgsl`, `scatter.wgsl`, `g2p.wgsl`, and `impulse.wgsl` own the particle index
  formula and partial-tile guard.
- `crates/fluid-lab/src/gpu/mod.rs`, `crates/fluid-lab/src/profiler/mod.rs`,
  `crates/fluid-lab/src/lib.rs`, and `web/panels.js` own truthful preflight and
  stats/panel reporting.
- `timing.rs` should not need new sections, but detailed mode must still route through
  the same dispatch helpers as coarse mode.
- Renderer files are measurement context only; do not add render decimation or high
  particle-count presets in this plan.

## Exit gate

- The particle dispatch audit identifies every shader/runtime path touched by tiled
  indexing.
- Tiled indexing is implemented consistently across mark, scatter U/V/W, G2P, and
  impulse, or the plan records the exact blocker.
- Manual slosh and scheduled wave-maker impulses submit valid work above the old 1D
  ceiling or are explicitly blocked with evidence.
- Wall-aware G2P behavior is preserved: static-wall stencil skipping, CFL clamp, wall
  friction, RK1 advect, and deterministic recovery are not changed by the tiling pass.
- Preflight still rejects impossible configurations before allocation/submission.
- Requested 8M no longer fails solely because of the old one-dimensional dispatch
  limit, or the remaining blocker is recorded with evidence.
- A fresh measurement matrix records requested count, estimated/actual particles,
  particle dispatch shape, scale status, frame percentiles, dropped time, GPU memory,
  timing source, and sorted GPU costs for each attempted row.
- `v1.9.0-particle-performance-followup.md` is updated with the selected optimization
  class or a reason no follow-up is ready.

## Discipline rules

- Do not remove preflight safety.
- Do not claim an interactive 8M target unless measurements support it.
- Do not expose high-count presets.
- Keep "higher legal ceiling" separate from "faster frame time" in reports.
- Do not update architecture or decisions docs until implementation is underway or
  shipped.
