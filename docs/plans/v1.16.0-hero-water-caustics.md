# v1.16.0 — Hero water: approximate caustics

> **SHIPPED (2026-06-09, branch `hero-water-14-18`).** Implemented as normal-gradient
> screen-space caustics: a half-res generation pass (reusing v1.15's `HeroParams.sun_direction`)
> + an additive receiver-composite pass inserted into the v1.12 render order *before* the
> water composite, so refraction sees lit caustics. All `render.hero.caustics.*` controls are
> Live; default OFF. Opus review caught two must-fix GPU bugs the build could not (scene_color
> read-write hazard → switched to additive ONE+ONE blend; eye-space-normal vs world-space-sun
> mismatch → eye_to_world transform). Valid DamBreak on/off A/B (EVAL echo present, no device
> errors); on-floor focused brightening reads as light, not noise — **low-medium confidence,
> still-frame; live session is the real gate.** Cost: `gpu.render_ms` +0.82 ms (1.21 on / 0.40
> off). Known watch-items for live review: subtlety of the effect; `mode`/`resolution_scale`/
> `blur_radius` settings are reserved/not-yet-wired (tooltips say so). Architecture/decisions
> doc migration deferred to the final consolidated pass (see
> [`hero-water-delivery-hub.md`](hero-water-delivery-hub.md)).

> Hero-water series plan 5 of 7. Inherits the *Cross-cutting decisions* in
> [`v1.12.0-hero-water-refraction.md`](v1.12.0-hero-water-refraction.md). **Depends on
> v1.12** — caustics need receivers (the textured floor/back wall) and a light direction,
> both established there (light dir shared with v1.15's environment).

## Mission

Cast moving focused-light patterns on the tank floor and back wall — the high-value
"this is transparent water in a tank" cue. Done when the dam-break capture shows caustics
that move with the surface and read as light focusing, not as noise, and intensity is
tunable live.

## Scope

**In:**
- A caustic target (half-res by default), generated by an **approximate normal-gradient**
  model: surface-normal focusing × light intensity × thickness visibility × receiver
  visibility (the source plan's first-implementation formula).
- Composite caustics onto the floor and back wall in the v1.12 `scene_color` (so the water
  refraction sees lit caustics behind it) or as a receiver overlay — pick one and document.
- Simple per-target history blend for stability (a stopgap until the full v1.18 temporal
  plan; keep the knob name compatible).
- Debug view: `Caustics`.

**Out:** physically-projected photon/refracted-light splatting (source plan's "better
implementation later"); side-wall caustics off by default; anything requiring a real
shadow map.

## Approach

Independent shader + target; reads the surface normal/thickness already produced by the
composite path and the procedural light direction from v1.15 (or a standalone sun if
v1.15 isn't in yet).

1. Caustic compute/fragment writing the half-res target from normal gradient + thickness.
2. Composite onto floor/back-wall receivers.
3. Blur + simple temporal blend.
4. Capture Scene 3 (dam break); tune intensity/blur; profiler.

## Watch-outs

- **"Looks fake" is the default failure.** Mitigate with: lower intensity, more blur,
  temporal smoothing, floor/back-wall only, drive by surface-normal variation. Budget
  time for tuning, not just wiring.
- **Ordering vs refraction:** if caustics are composited into `scene_color` *before* the
  water pass, the refracted background correctly shows caustics bent through the water —
  but they must be written before the scene→swapchain blit. Get the pass order right
  (this is the same ordering subtlety v1.12 introduced).
- Half-res target + temporal history needs ping-pong storage; account for the memory in
  `gpu-resources.md`.
- No readback; the caustic target is GPU-only.

## Settings (Water tab → Caustics group, all Live)

Mirror `[render.hero_water.caustics]`: `caustics.enabled`, `mode` (enum),
`resolution_scale`, `intensity`, `focus_strength`, `thickness_scale`, `floor_enabled`,
`back_wall_enabled`, `side_walls_enabled`, `blur_radius`, `temporal_enabled`,
`temporal_alpha`, `motion_scale`, `max_intensity`.

- Scene 3 capture: caustics move with the surface, read as focused light not noise.
- Intensity/blur sliders change the look live.

## Concrete implementation (orchestrator-refined, 2026-06-09)

Code-grounded against the committed `hero-water-14-18` baseline (v1.12 refraction +
v1.13 foam/diffuse + v1.15 environment/skybox, all in tree). Source-of-truth
precedence applied: runtime Rust/WGSL > this plan > architecture docs > the ChatGPT
draft above.

### 1. Real light-direction source

The sun already exists and is shared — do NOT invent a standalone sun.

- `settings::HeroParams.sun_direction: [f32; 3]` + `sun_intensity: f32`
  (`app/crates/fluid-lab/src/settings/mod.rs` lines 207–208), assembled in
  `Registry::hero_params()` (lines 1626–1631) from the **registered** settings
  `render.hero.sun_dir_x` / `render.hero.sun_dir_y` / `render.hero.sun_dir_z`
  (defaults 0.4 / 0.7 / 0.5, Advanced group) and `render.hero.sun_intensity`
  (default 1.2). World-space; normalized in WGSL.
- It is already mirrored into the GPU in two places by `set_hero_params`
  (`gpu/mod.rs:683`): the composite `HeroUniform.sun` field
  (`gpu/composite.rs:103-108`, WGSL `hero.sun` in `composite.wgsl:16`) and the
  skybox `SkyUniform.sun` (`gpu/skybox.rs:177-182`). `env.wgsl::env_sample` takes
  `sun: vec4<f32>` = `(dir.xyz, intensity)`.

The caustic shader reuses this exact `(sun_direction, sun_intensity)` — wire it into
the new caustic uniform from the same `HeroParams` fields inside `set_hero_params`,
so caustic light stays coherent with the sky/specular sun. No new light concept.

### 2. Pass-order plan (exact insertion in `gpu/mod.rs::render`, `RenderMode::Water` arm)

The committed Water arm runs these passes in one `frame` encoder (lines ~1086–1289):

1. **"hero scene prepass"** (1089) — skybox → environment (floor/back/left wall) →
   wireframe, into `scene_color_view` (Rgba16Float HDR) + `scene_depth_view`
   (R16Float eye-distance) + `depth_view`. THIS is the receiver buffer.
2. "water thickness pass" (1133) — particles → `thickness/nearest_z/whitewater`.
3. "water smooth x pass" (1185) → `smooth_z_ping_view`.
4. "water smooth y pass" (1209) → `smooth_z_view` (the smoothed front-surface Z the
   composite differentiates into the water normal).
5. "water composite pass" (1234) — `composite.draw` reads `scene_color` (refracted
   bg, normal-driven UV tap) + `scene_depth` (depth guard) and writes the final pixel
   to the swapchain `view`. **This IS the scene→swapchain step** (REPLACE blend,
   `composite.rs:336-340`); there is no separate blit.
6. "diffuse particle pass" (1261) — foam over the composite, if active.

**Insertion points (two new passes):**

- **(A) Caustic generation pass** — inserted AFTER pass 4 ("water smooth y") and
  BEFORE the new composite-into-scene pass (B). It needs the smoothed water normal
  (`smooth_z_view`), water `thickness_view`, and `sun_direction`. Renders the
  half-res `caustic_view` (R16Float). Fullscreen-triangle fragment pass, same idiom
  as `smoothing` passes.
- **(B) Caustic→receiver composite pass** — inserted AFTER (A) and BEFORE pass 5
  ("water composite pass"), reading `scene_depth_view` + `caustic_view` and writing
  back into `scene_color_view`. Because it mutates `scene_color` *before* the water
  composite samples it, the water's refracted-background tap (composite.wgsl:170,
  `textureSample(scene_color_tex, …, refract_uv)`) sees lit caustics bent through the
  liquid — satisfying the "refraction must see lit caustics" requirement — and it is
  written before the composite blits scene→swapchain in pass 5.

  Receiver gating: there is no `kind` G-buffer after the prepass (environment.wgsl
  only stores color+eye), so (B) reconstructs the world hit point from
  `scene_depth_view` (positive eye distance) along the per-pixel eye ray, using the
  composite's existing `tan(fov_y/2)` + `cam.eye_to_world` rotation (already uploaded
  via `composite.set_camera`/`skybox.set_camera`; reuse the same `eye_to_world`
  matrix and `tank_lo/tank_hi` already known to the renderer). Floor receiver =
  world `y` near `tank_lo.y`; back-wall receiver = world `z` near `tank_lo.z`. This
  reuses `fluid.tank_bounds()` (already called at `gpu/mod.rs:236,424`).

  Alternative considered & rejected for v1.16: writing a `kind`/receiver-id channel
  from the environment pass would be cleaner but adds a 4th prepass attachment and
  touches environment.rs/.wgsl — out of scope; revisit if reconstruction proves
  fiddly.

(B) blends additively into `scene_color` (load existing color, add caustic*sun), so
geometry that is not a receiver is untouched. The temporal history blend (knob below)
lives inside (A) via a ping/pong `caustic_history_view` pair.

### 3. Settings ids (codebase convention: `render.hero.caustics.*`, all Live) + HeroParams fields

The `render.hero.` prefix is batch-routed Live with no per-id plumbing: `lib.rs:618`
`if id.starts_with("render.hero.")  { self.gpu.set_hero_params(self.settings.hero_params()); }`.
So every caustics knob MUST be `render.hero.caustics.*` to ride that path with zero
new dispatch code — the renderer just reads new `HeroParams` fields in a new
`caustic_uniform(hero)` (mirroring `hero_uniform`). Register each in the
`Registry::default()` table (Category::Water, ApplyClass::Live, Default/Advanced
group) and read it in `hero_params()`; add each id to the Water-tab Live assertion
list (settings/mod.rs ~1992).

Setting ids → HeroParams fields:

### 4. Normal-gradient caustic approach (half-res, blur, temporal)

Per-pixel in the half-res generation pass (A), fullscreen triangle over
`caustic_view`:

- Sample the smoothed front-surface depth `smooth_z` and compute the water normal
  with the SAME finite-difference + world-per-pixel scaling already in
  `composite.wgsl::water_normal` (lines 61–86) — factor it or duplicate the math so
  caustics and refraction agree on the normal.
- Focusing term: caustic brightness ∝ how much the surface *converges* light. Use the
  screen-space divergence of the refraction offset, i.e. the Laplacian / gradient
  magnitude of the normal field (reuse the `curv` Laplacian already computed at
  composite.wgsl:111-121): `focus = 1 + focus_strength * curvature_convergence`, with
  the sign favoring concave (focusing) regions. Multiply by
  `max(dot(n, sun_dir), 0)` so light only focuses where the sun actually hits the
  surface.
- Visibility terms (the source plan's first-implementation formula):
  `caustic = focus * sun_intensity * thickness_visibility * receiver_visibility`,
  where `thickness_visibility = 1 - exp(-thickness * caustics_thickness_scale)`
  (only lit where there is water above), and `receiver_visibility` is applied later
  in (B) by the floor/back-wall world-position gate (not in (A)).
- Clamp to `caustics_max_intensity`; scale by `caustics_intensity`. `motion_scale`
  feeds a slight UV advection by the normal's xy so the pattern slides with the
  surface rather than sitting static.
- **Blur**: a small separable blur (radius `caustics_blur_radius`) — either two extra
  half-res passes like the existing `smoothing` x/y split, or a cheap in-shader 3x3.
  Prefer reusing the smoothing two-pass idiom for a real Gaussian; it is the
  established pattern (`gpu/smoothing.rs`).
- **Temporal blend** (stopgap for v1.18): ping/pong `caustic_history_view` pair;
  output = `mix(history, current, caustics_temporal_alpha)` when
  `caustics_temporal_enabled`. The knob `render.hero.caustics.temporal_alpha` is named
  to survive into the v1.18 unified temporal pass.

GPU-only, no readback (consistent with the "no readback" rule). The caustic target +
history are new GPU memory (half-res R16Float ×2 history + 1 working = ~3 half-res
R16 targets) — record sizes in `gpu-resources.md` at ship.

### 6. Risks / watch-outs (refined)

- **Receiver reconstruction accuracy.** Reconstructing world pos from R16Float eye
  distance + eye ray is the riskiest bit; R16 precision at distance may make the
  floor/back-wall plane test jittery. Mitigation: use a tolerance band on
  `y≈tank_lo.y` / `z≈tank_lo.z` proportional to depth; if too noisy, fall back to a
  dedicated `kind` channel in a v1.16.1.
- **`resolution_scale` Live-ness.** Changing target resolution means reallocating the
  half-res targets, which the `render.hero.*` Live batch route does not currently do
  (it only re-uploads uniforms). Handle by detecting a resolution_scale change inside
  `set_hero_params` and recreating the caustic targets there (cheap, no sim reset),
  OR document it as effectively Reset. Prefer the in-place realloc to honor "all Live".
- **"Looks fake" default failure** (from the draft) — budget tuning time; ship with
  conservative defaults (moderate intensity, blur 1.5, temporal_alpha 0.15, floor +
  back-wall only, side walls off).
- **Pass-order correctness.** (B) MUST write `scene_color` before pass 5 reads it;
  putting (A)+(B) after smooth-y and before the water composite is the only ordering
  where refraction sees lit caustics AND they reach the swapchain via the composite's
  REPLACE blit. Do not move them after the composite.
- **Timing.** The composite pass currently carries `render_end_writes` when no diffuse
  pass follows (mod.rs:1248). Adding caustic passes before the composite does not move
  `render_end`; just confirm the new passes have `timestamp_writes: None` so the
  fine-section / render_begin..render_end accounting is unchanged. Never fabricate the
  `gpu.render_ms` delta — measure it from a real capture at gate time.
