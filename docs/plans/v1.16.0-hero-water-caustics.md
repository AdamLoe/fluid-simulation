# v1.16.0 — Hero water: approximate caustics

> Hero-water series plan 5 of 7. Inherits the *Cross-cutting decisions* in
> [`v1.12.0-hero-water-refraction.md`](v1.12.0-hero-water-refraction.md). **Depends on
> v1.12** — caustics need receivers (the textured floor/back wall) and a light direction,
> both established there (light dir shared with v1.15's environment).

## Mission

Cast moving focused-light patterns on the tank floor and back wall — the high-value
"this is transparent water in a tank" cue. Done when the dam-break capture shows caustics
that move with the surface and read as light focusing, not as noise, and intensity is
tunable live.

## Scope

**In:**
- A caustic target (half-res by default), generated by an **approximate normal-gradient**
  model: surface-normal focusing × light intensity × thickness visibility × receiver
  visibility (the source plan's first-implementation formula).
- Composite caustics onto the floor and back wall in the v1.12 `scene_color` (so the water
  refraction sees lit caustics behind it) or as a receiver overlay — pick one and document.
- Simple per-target history blend for stability (a stopgap until the full v1.18 temporal
  plan; keep the knob name compatible).
- Debug view: `Caustics`.

**Out:** physically-projected photon/refracted-light splatting (source plan's "better
implementation later"); side-wall caustics off by default; anything requiring a real
shadow map.

## Approach

Independent shader + target; reads the surface normal/thickness already produced by the
composite path and the procedural light direction from v1.15 (or a standalone sun if
v1.15 isn't in yet).

1. Caustic compute/fragment writing the half-res target from normal gradient + thickness.
2. Composite onto floor/back-wall receivers.
3. Blur + simple temporal blend.
4. Capture Scene 3 (dam break); tune intensity/blur; profiler.

## Watch-outs

- **"Looks fake" is the default failure.** Mitigate with: lower intensity, more blur,
  temporal smoothing, floor/back-wall only, drive by surface-normal variation. Budget
  time for tuning, not just wiring.
- **Ordering vs refraction:** if caustics are composited into `scene_color` *before* the
  water pass, the refracted background correctly shows caustics bent through the water —
  but they must be written before the scene→swapchain blit. Get the pass order right
  (this is the same ordering subtlety v1.12 introduced).
- Half-res target + temporal history needs ping-pong storage; account for the memory in
  `gpu-resources.md`.
- No readback; the caustic target is GPU-only.

## Settings (Water tab → Caustics group, all Live)

Mirror `[render.hero_water.caustics]`: `caustics.enabled`, `mode` (enum),
`resolution_scale`, `intensity`, `focus_strength`, `thickness_scale`, `floor_enabled`,
`back_wall_enabled`, `side_walls_enabled`, `blur_radius`, `temporal_enabled`,
`temporal_alpha`, `motion_scale`, `max_intensity`.

## Exit gate

- WSL build + `cargo test --lib` green.
- Scene 3 capture: caustics move with the surface, read as focused light not noise.
- Intensity/blur sliders change the look live.
- `gpu.render_ms` delta + caustic-target memory recorded.
