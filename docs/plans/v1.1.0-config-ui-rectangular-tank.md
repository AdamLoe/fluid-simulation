# v1.1.0 — config-UI polish + rectangular tank

## Mission

One combined release that (a) polishes the config UX — a camera-zoom default,
reworded two-part tooltips, a per-setting reset button, and a stripped-down
header — and (b) lifts the long-standing cubic-tank assumption so the tank can
be a **rectangular box** with independent X/Y/Z dimensions. Done = all five
items land, the WASM builds, host tests pass, and a real-GPU capture shows the
non-cube tank running with the new UI. The bounce/float **objects** feature
(originally item #5) is explicitly **deferred** to a later release (see
[Deferred](#deferred)).

## Scope

**In scope (the five items):**

1. **Camera zoom default (#1).** New `camera.distance` Live setting (F32,
   default `6.0`, range `[2.0, 40.0]` to match the existing wheel-zoom clamp).
   Restored on `reset()`, applied live from the panel. Mirrors the existing
   `camera.rot_x/y/z` pattern exactly.
2. **Tooltip reword (#2).** Rewrite all setting tooltips into a consistent
   two-part form — a plain-language *effect* sentence first, then the
   *technical* detail — including the new settings added by items #1 and #4.
   Single `tooltip` string per setting; **no schema change** (the panel renders
   the string as-is, splitting on a `—` separator for light visual structure).
3. **Per-setting reset button (#3).** Each control in the config panel gets a
   small reset-to-default icon button to its right; clicking calls
   `app.set_setting(id, default)` (the `default` already ships in
   `config_json`). Reset-class settings reuse the existing "needs reset" cue.
4. **Rectangular tank (#4).** Replace the fixed `[-1,1]³` cube with a
   rectangular box driven by three Reset-class **per-axis cell-count** settings
   (`grid.res_x/y/z`, U32, default 64, range 16–128), which replace the single
   `grid.resolution`. **Cell size stays uniform** — a module constant `h = 2/64`
   — so the domain extent on each axis is `n_axis · h` (all-64 reproduces the
   exact current `[-1,1]³` cube) and the **pressure operator stays isotropic**
   (`1/h²`, no `hx/hy/hz`). `GridDims` is already `nx/ny/nz`-aware and
   `SceneConfig.grid_resolution` is already a `UVec3`. The work is therefore:
   rewrite the ~10–12 WGSL shaders that hardcode a single `N` cell-decomposition
   (`c%n,c/n,c/(n²)` → `c%nx,(c/nx)%ny,c/(nx·ny)`; `vec3(n,n,n)` face dims →
   per-axis) and `cells = n³` → `nx·ny·nz`; decouple the hardcoded domain
   extent (`2.0`) / origin (`-1.0`) from the per-axis values; parameterize the
   wireframe tank to an arbitrary AABB. Buffer counts derive from
   `nx/ny/nz`.
5. **Header simplification (#6).** Strip the top `#toolbar` to: a "Fluid Lab"
   title (top-left) and the buttons Pause / Reset / Mesh / Settings (top-left),
   with **no background and no border**. Remove the Step button, both hint
   texts, and the spacer. Rename the panels toggle to "Settings". The bottom
   `#mode-bar` interaction buttons and `1`–`4` keyboard shortcuts are **not**
   in the top bar and are left untouched.

- `app/Cargo.toml` version bump to `1.1.0`.

**Out of scope:**

- **Objects that bounce/float (#5)** — deferred; see [Deferred](#deferred).
- **Anisotropic-h physics tuning.** The rectangular refactor keeps the existing
  FLIP/PIC math; we do not re-tune CFL/FIXED_SCALE for extreme aspect ratios.
  Constrain extent ranges so cell counts stay within current memory/perf
  envelope (e.g. each axis maps to a sane cell count at the chosen base
  resolution). Extreme box ratios are a follow-up perf/quality concern.
- **Vite/TS path.** All UI work targets the canonical static path
  (`static.html` + `main.js` + `panels.js`) only.
- **New tooltip schema.** Tooltips stay a single string; no `config_json`
  field changes for item #2.

## Approach

Two streams run **in parallel**: the web stream (W) is pure JS/HTML/CSS and
shares no files with Rust; the Rust stream (R) is sequenced internally because
items #1, #4, and #2 all edit `settings/mod.rs` and `lib.rs` and must not race
on those files.

### Stream W — web UI (parallel, owns the web files)

Owned files: `app/web/static.html`, `app/web/main.js`, `app/web/panels.js`.
Covers items **#3** and **#6**. No Rust dependency — new settings from Stream R
appear automatically because the panel is data-driven off `config_json`.

- **#3 per-setting reset:** add a reset icon button in `panels.js →
  buildSettingRow` and `buildEnumRow` (after the slider/number/select), styled
  with a new `.cfg-reset-btn` class in `static.html`. On click:
  `app.set_setting(s.id, s.default)`, update the control, and for reset-class
  settings fire the same "needs reset"/`app.reset()` path the panel already uses.
- **#6 header:** edit the `#toolbar` markup in `static.html` (drop Step, the two
  hint spans, the spacer; rename `#btn-panels` → `#btn-settings`, label
  "Settings"); remove `background`/`border-bottom` from the `#toolbar` CSS;
  update `main.js`/`panels.js` references from `btn-panels` to `btn-settings`
  (the existing `setPanelsVisible` toggle is reused as the Settings toggle).

### Stream R — Rust (sequenced: R1 → R2 → R3)

- **R1 — camera zoom (#1).** `settings/mod.rs`: add `camera.distance` Setting +
  `camera_distance()` accessor. `camera.rs`: add `set_distance(d)` (clamps
  `[2,40]`). `lib.rs`: apply in `create()` and `reset()`, add the Live dispatch
  arm in `set_setting`. Small, lands first to get the registry edit out of the
  way.
- **R2 — rectangular tank (#4), the heavy refactor.** Cell size stays a uniform
  scalar `h = 2/64`; only cell counts vary per axis (see Scope #4). Touches:
  - `settings/mod.rs` — replace `grid.resolution` with `grid.res_x/y/z` (U32,
    default 64, 16–128, Reset) in the `Grid` category; accessors
    `grid_res_x/y/z()` (drop/redirect `grid_resolution()`; update its call
    sites, e.g. `stats_json`).
  - `sim/mod.rs` — confirm `GridDims{nx,ny,nz,h,origin}` math is axis-correct;
    keep scalar `h`. Add host tests for non-cubic indexing (`world_to_cell`,
    `cell_center_world`, face indexers) with `nx≠ny≠nz`.
  - `gpu/fluid.rs` — expand `Params` to carry `nx/ny/nz` (replace single `n` in
    `dims`) and per-axis domain extent/origin in `origin`/a new field; keep
    `h`/`inv_h` scalar; fix `generate_particles` `to_world` and wall bounds
    (drop hardcoded `2.0`/`-1.0`, use `origin`+`n_axis·h`); recompute all face
    buffer counts from `nx/ny/nz`.
  - **WGSL shaders** (rewrite single-`N` cell decomposition `c%n, c/n, c/(n²)`
    → `c%nx, (c/nx)%ny, c/(nx*ny)` and `vec3(n,n,n)` face dims → per-axis):
    `divergence`, `gradient`, `scatter`, `g2p`, `boundaries`, `mark`,
    `classify`, `forces`, `pressure` (RBGS), `cg_spmv`, and the `cells = n³` ones
    (`cg_init/dir/reduce/reduce_final/update`).
  - `gpu/renderer.rs` — parameterize `tank_wireframe(lo, hi)` for an arbitrary
    AABB instead of `[-1,1]³`.
  - `scene/mod.rs` — keep `LiquidBlock` in normalized `[0,1]³`; map to world via
    the per-axis extents (not hardcoded `2.0`).
  - `lib.rs` — accessors/wiring; `reset()`/`recreate_fluid` pick up the extents.
  - **Invariants to preserve:** integer/fixed-point P2G (`scatter`→`normalize`);
    boundary cells still all-Solid so every Liquid cell stays interior (no
    bounds-check additions); every compute shader still references `params`
    (binding 0) or uses an explicit BGL; CG stays the default solver.
- **R3 — tooltip reword (#2).** Final consolidating pass over **all** settings
  in `settings/mod.rs` (now including `camera.distance` and `tank.size_*`):
  two-part "effect — technical" strings. Pure content; lands last so no setting
  is missed.

Stream W and Stream R run concurrently. Within R, R1 → R2 → R3 serialize on
`settings/mod.rs`/`lib.rs`.

- `wsl.exe -d Ubuntu-24.04 -- bash -lc 'cd /home/adamg/fluid-simulation/app && cargo test --lib'` — host reference tests pass, including new non-cubic / anisotropic-`h` `GridDims` indexing tests and the existing pressure/MC tables.
- Real-GPU capture via `tools/capture.mjs` against `http://localhost:5184/static.html` (the acceptance signal that can't be faked), showing:
  - the tank rendered as a **non-cube box** when `tank.size_*` differ, fluid filling and settling correctly (no leaks through walls, divergence/pressure stable);
  - the simplified header (title + Pause/Reset/Mesh/Settings, no background/border);
  - per-setting reset buttons present and restoring defaults;
  - `camera.distance` changing the initial zoom and surviving Reset.
- Console log shows the healthy boot (`navigator.gpu present: true`, smoke-test PASS, `fluid init: …`) — not the `#unsupported` overlay.

## Discipline rules

- **Rectangular tank is Reset-class.** `grid.res_*` (and any grid change) require
  `recreate_fluid` (buffer realloc); they must not be forced Live.
- **No determinism regressions.** Do not "simplify" P2G to float accumulation
  while touching shaders; the integer atomic path is load-bearing.
- **One WGSL shader family at a time** during R2, rebuilding + capturing between
  families — a silent naga unused-binding drop or an off-by-one in cell
  decomposition is easiest to localize incrementally.

## Deferred

**Objects that bounce/float (original #5)** — deferred to a later, dedicated
release. The audit established two tiers: **Tier A** (CPU rigid body with
geometric buoyancy + drag + wall-bounce, rendered as a cube/sphere mesh,
optional weak fluid push via the existing impulse pass) is low-risk and touches
no pressure-solver/readback/determinism code; **Tier B** (object as moving solid
cells in the pressure projection) breaks the "every Liquid cell is interior /
no bounds checks" CG invariant and is multi-week with uncertain solver
stability. When picked up, this belongs in `decisions/scope.md` (which already
defers "dynamic solids") and likely starts from Tier A. Add it to
[`roadmap.md`](roadmap.md) at ship time of this plan.
