# v1.0.1 ‚Äî FPS, camera rotation, and sun direction sliders

## Mission

Start explicit project versioning at **1.0.1** and ship three groups of new
Live-class config sliders: an FPS target, initial camera orientation (X/Y/Z
rotation), and sun direction (X/Y/Z) for the mesh lit surface. Every slider
round-trips through the existing settings registry and panel with zero new
JS control code required.

## Scope

**In scope:**
- `Cargo.toml` version bump to `1.0.1`.
- `render.fps_target` (U32 0‚Äì240, default 60, Live) ‚Äî throttles the JS rAF
  loop.
- `camera.rot_x/y/z` (F32 ‚àíœÄ..œÄ, Live) ‚Äî sets initial camera pitch/yaw/roll;
  also applied immediately while Live so the slider is interactive.
- `render.sun_x/y/z` (F32 ‚àí2..2, Live) ‚Äî sun direction for the mesh lit
  surface (`mesh.wgsl`), normalized in the shader.
- New `Category::Camera` enum variant so camera settings group separately in
  the panel.
- Extend `MeshCamera` uniform (WGSL struct + Rust mirror) to carry `sun_dir`.
- `fps_target()` accessor exported via `wasm_bindgen` so `main.js` can read it.
- Camera `roll` field + yaw/pitch/roll setters in `OrbitCamera`.
- On `reset()`, camera restores to the settings-defined rotation values (not
  hardcoded 0.6/0.4).

**Out of scope:**
- Sun shading for particles or slice views (no lighting there today).
- Camera distance slider (separate concern; can be added later).
- Any UI-framework changes; the panel renders the new settings automatically
  from `config_json`.

## Approach

Eight files change; all are independent and can be done sequentially in one
pass:

1. `app/Cargo.toml` ‚Äî version bump.
2. `app/crates/fluid-lab/src/settings/mod.rs` ‚Äî `Category::Camera`, 7 new `Setting` entries,
   typed accessors (`fps_target`, `camera_rot_x/y/z`, `sun_x/y/z`).
3. `app/crates/fluid-lab/src/camera.rs` ‚Äî add `roll: f32`, update `orientation()`, add
   `set_yaw/set_pitch/set_roll`.
4. `app/crates/fluid-lab/src/gpu/shaders/mesh.wgsl` ‚Äî extend `Camera` struct with
   `sun_dir: vec4<f32>`; use `normalize(cam.sun_dir.xyz)` as `L`.
5. `app/crates/fluid-lab/src/gpu/mesh.rs` ‚Äî extend `MeshCamera` struct, update `update_camera`
   signature to accept `sun_dir: glam::Vec3`.
6. `app/crates/fluid-lab/src/gpu/mod.rs` ‚Äî add `sun_dir: glam::Vec3` field, `set_sun_dir`,
   pass to `mesh.update_camera` in `render()`.
7. `app/crates/fluid-lab/src/lib.rs` ‚Äî Live dispatch for all 7 settings; `fps_target()`
   wasm_bindgen accessor; update `reset()` to restore camera from settings.
8. `app/web/main.js` ‚Äî throttle rAF loop by `app.fps_target()`.

## Exit gate

- `wsl.exe -d Ubuntu-24.04 -- bash -lc 'cd /home/adamg/fluid-simulation/app && cargo build --target wasm32-unknown-unknown'` ‚Äî clean build.
